버그를 수정한다.

## 입력
- 증상: $ARGUMENTS

## 절차
1. 에러 메시지/증상을 분석하라
2. 관련 코드를 explore로 탐색하라
3. 원인을 파악하고 수정 범위를 결정하라
4. 수정 후 관련 테스트가 통과하는지 확인하라
5. 같은 유형의 버그가 다른 곳에도 있는지 검색하라

## 예시

### /fix 로그인 후 리다이렉트 안 됨
1. 에러/증상 분석: "로그인 성공 후 / 페이지에 머묾"
2. 탐색: auth 관련 코드, redirect 로직 검색
3. 원인: `router.push`가 `await` 없이 호출 → 상태 업데이트 전 리다이렉트
4. 범위: S (auth.ts 파일 1개)
5. 직접 수정 후 확인
