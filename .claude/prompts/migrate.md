# /migrate — 마이그레이션

DB 스키마 변경, 의존성 메이저 업그레이드, 프레임워크 마이그레이션 등 위험한 전환 작업을 안전하게 수행한다.

## 인자
- `[대상]`: 마이그레이션 대상 (예: "TypeORM 0.3→0.4", "React 18→19", "users 테이블에 role 컬럼 추가")

## 실행 절차

1. **영향 분석**
   - 변경이 영향을 미치는 파일/모듈 목록
   - 하위 호환성이 깨지는 부분 식별
   - 롤백 가능 여부 확인

2. **마이그레이션 계획 제시**
   ```
   ## Migration Plan: [대상]

   영향 범위: N개 파일
   하위 호환: 깨짐 / 유지
   롤백: 가능 (방법) / 불가능 (이유)

   Steps:
   1. [준비 단계]
   2. [실행 단계]
   3. [검증 단계]
   ```
   - **사용자 승인 후 진행**

3. **단계별 실행**
   - 한 단계씩 실행하고 검증
   - DB 마이그레이션이면: 마이그레이션 파일 생성 → 테스트 DB에서 실행 → 확인
   - 의존성 업그레이드면: 업그레이드 → 빌드 → 테스트 → breaking change 수정

4. **검증 체크리스트**
   - [ ] 빌드 성공
   - [ ] 기존 테스트 전체 통과
   - [ ] 새 기능/변경 동작 확인
   - [ ] 롤백 절차 문서화 (필요 시)

## 마이그레이션 유형별 주의사항

| 유형 | 핵심 위험 | 필수 체크 |
|------|----------|----------|
| DB 스키마 | 데이터 손실, 다운타임 | 백업 확인, 롤백 SQL 준비 |
| 의존성 업그레이드 | Breaking API 변경 | CHANGELOG/migration guide 읽기 |
| 프레임워크 전환 | 대규모 코드 변경 | 점진적 전환 (strangler fig) |
| 설정/인프라 | 서비스 중단 | 카나리 배포, 헬스체크 |

## 규칙
- **반드시 계획을 먼저 보여주고 승인받는다.** 마이그레이션은 되돌리기 어렵다.
- **한 번에 하나.** 여러 마이그레이션을 동시에 하지 않는다.
- **테스트 없이 진행하지 않는다.** 테스트가 없으면 먼저 작성.
- **롤백 불가능한 경우 명시적으로 경고한다.**
- 위험도가 높으면 `/careful` 모드를 자동 활성화한다.
