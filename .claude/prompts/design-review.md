# /design-review — 시각적 디자인 QA

라이브 사이트 또는 로컬 서버의 시각적 품질을 점검한다.
간격, 정렬, 타이포그래피 계층, 색상 일관성, 인터랙션 상태를 검토하고 수정한다.

## 인자
- `[URL]`: 검토할 URL (필수)

## 실행 절차

1. **페이지 스크린샷 캡처** (browse 도구)
   - 데스크톱, 태블릿(768px), 모바일(375px) 3가지 뷰포트

2. **5가지 디자인 차원 점검**

   ### 간격 & 정렬 (Spacing)
   - 요소 간 간격이 일관적인가?
   - 정렬이 맞는가? (좌측, 중앙, 그리드)
   - 패딩/마진이 의도와 다른 곳이 있는가?

   ### 타이포그래피 계층 (Typography)
   - 제목 → 본문 → 캡션 크기 계층이 명확한가?
   - 폰트 굵기, 줄 간격이 일관적인가?
   - 텍스트가 잘리거나 넘치는 곳이 있는가?

   ### 색상 & 대비 (Color)
   - 색상 팔레트가 일관적인가?
   - 텍스트-배경 대비가 충분한가? (WCAG AA 기준)
   - hover/active 상태 색상이 적절한가?

   ### 인터랙션 상태 (States)
   - 빈 상태 (empty state)가 처리되어 있는가?
   - 로딩 상태가 표시되는가?
   - 에러 상태 UI가 있는가?
   - hover, focus, disabled 상태가 구분되는가?

   ### 반응형 (Responsive)
   - 모바일에서 깨지는 레이아웃이 있는가?
   - 터치 타겟이 44px 이상인가?
   - 가로 스크롤이 발생하는가?

3. **문제 발견 시 수정**
   - 각 문제를 스크린샷과 함께 설명
   - CSS/컴포넌트를 직접 수정
   - 수정 전/후 스크린샷 비교

4. **결과 보고**
   ```
   ## Design Review — [URL]

   Score: X/10

   | 차원 | 점수 | 이슈 |
   |------|------|------|
   | 간격 & 정렬 | X/10 | ... |
   | 타이포그래피 | X/10 | ... |
   | 색상 & 대비 | X/10 | ... |
   | 인터랙션 상태 | X/10 | ... |
   | 반응형 | X/10 | ... |

   수정: N건
   ```

## 규칙
- browse 도구가 필요 (`--tools=browse`로 설치)
- 기능 변경은 하지 않는다 (스타일/레이아웃만)
- 수정마다 원자적 커밋 (한 이슈 = 한 커밋)
