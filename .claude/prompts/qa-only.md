# /qa-only — 리포트 전용 QA

브라우저 기반 QA 테스트를 실행하되, 버그를 **보고만** 하고 수정하지 않는다.
/qa의 리포트 전용 모드.

## 인자
- `[URL]`: 테스트할 URL (필수)

## 실행 절차

1. **대상 파악**
   - URL을 브라우저로 열고 페이지 구조 파악
   - 피처 브랜치면 `git diff`로 변경 관련 페이지 우선 테스트

2. **테스트 수행** (browse 도구 사용)
   - 페이지 로딩, 네비게이션, 인터랙션
   - 콘솔 에러, 네트워크 에러
   - 반응형 레이아웃 (모바일/태블릿/데스크톱)
   - 폼 입력, 버튼 클릭, 모달 등 주요 인터랙션

3. **리포트 출력**
   ```
   ## QA Report — [URL]

   Health Score: X/10

   ### Critical
   - [버그 설명] (스크린샷/재현 경로)

   ### High
   - [버그 설명]

   ### Medium
   - [버그 설명]

   ### Low / Cosmetic
   - [버그 설명]
   ```

## 규칙
- **코드를 수정하지 않는다.** 보고만 한다.
- 스크린샷이 가능하면 첨부
- 재현 경로(클릭 순서)를 명시
- /qa와 동일한 테스트를 하되 수정 단계만 생략
