# /explain — 코드 설명

코드베이스의 특정 부분을 설명한다. 온보딩, 인수인계, 코드 이해에 사용.

## 인자
- `[대상]`: 설명할 대상 (파일, 모듈, 함수, 아키텍처, 흐름 등)

## 실행 절차

1. **대상 파악**
   - 인자가 파일이면 해당 파일 읽기
   - 인자가 모듈/개념이면 관련 파일 탐색
   - 인자가 흐름이면 ("로그인 흐름", "결제 프로세스") 관련 코드 추적

2. **구조적 설명**

   ### 한 줄 요약
   이 코드가 **무엇을 하는지** 한 문장으로.

   ### 왜 존재하는가
   이 코드가 **왜 필요한지**, 어떤 문제를 해결하는지.

   ### 어떻게 동작하는가
   핵심 흐름을 단계별로 설명. ASCII 다이어그램 사용.
   ```
   [입력] → [처리 A] → [처리 B] → [출력]
   ```

   ### 주요 의존성
   이 코드가 의존하는 다른 모듈/서비스.

   ### 주의할 점
   함정, 엣지 케이스, "왜 이렇게 했는지" 궁금할 만한 부분.

3. **출력 수준 조절**
   - 짧은 대상 (단일 함수): 간결하게
   - 넓은 대상 (모듈, 아키텍처): 상세하게 + 다이어그램

## 규칙
- **코드를 수정하지 않는다.** 읽기 전용.
- 주관적 판단("이 코드는 나쁘다")보다 객관적 설명에 집중
- 사용자의 수준에 맞게 설명 (초보자에게는 비유, 전문가에게는 구현 디테일)
