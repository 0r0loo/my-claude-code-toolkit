# /document-release — 배포 후 문서 갱신

코드 변경 후 README, CHANGELOG, CLAUDE.md를 diff와 대조하여 자동 갱신한다.
배포 체크리스트의 문서 업데이트를 자동화한다.

## 인자
- (없음): 최근 커밋 기준
- `[커밋범위]`: 특정 범위 (예: `v1.12.0..HEAD`)

## 실행 절차

1. **변경 범위 파악**
   ```bash
   git log --oneline ${RANGE:-HEAD~5..HEAD}
   git diff ${RANGE:-HEAD~5..HEAD} --stat
   ```

2. **문서 읽기**
   현재 README.md, CHANGELOG.md, `.claude/CLAUDE.md`를 읽는다.

3. **대조 및 갱신**

   ### README.md
   - 구조도(`## 구조`)가 현재 파일 구조와 일치하는지 확인
   - 설치 방법, 옵션 테이블이 실제 install.sh와 일치하는지 확인
   - 새 기능/스킬이 추가되었으면 해당 섹션에 반영

   ### CHANGELOG.md
   - 최신 버전 섹션에 변경사항이 누락 없이 기록되었는지 확인
   - 커밋 메시지를 기반으로 누락된 항목 추가
   - 문체 통일: `- **FEAT**: ...`, `- **FIX**: ...`, `- **REFACTOR**: ...`

   ### .claude/CLAUDE.md
   - 문서 참조 가이드(섹션 5)가 실제 파일과 일치하는지 확인
   - 새 에이전트/스킬/프롬프트가 추가되었으면 참조 목록에 반영
   - install.sh 구조 설명이 최신인지 확인

4. **결과 보고**
   ```
   ## /document-release 결과
   - README.md: [변경 사항 또는 "최신 상태"]
   - CHANGELOG.md: [변경 사항 또는 "최신 상태"]
   - CLAUDE.md: [변경 사항 또는 "최신 상태"]
   ```

## 규칙
- **코드는 건드리지 않는다.** 문서 파일만 수정한다.
- **기존 문체를 따른다.** 새로 추가하는 항목은 주변 항목의 스타일과 맞춘다.
- **추측하지 않는다.** diff에 없는 내용을 임의로 추가하지 않는다.
- 수정 후 `git diff`로 변경 내역을 보여준다.
