위험 명령을 실행 전에 검토한다.

## 입력
- $ARGUMENTS가 없으면 → 세션 모드 활성화
- $ARGUMENTS가 있으면 → 단일 명령 즉시 검토

---

## 모드 1: 세션 모드 (`/careful` 단독)

이후 이 대화에서 아래 **위험 명령 목록**에 해당하는 명령이 나오면,
실행 전에 반드시 위험도 평가와 대안을 제시하고 사용자 확인을 받는다.

```
⚠️ /careful 모드 활성화
이후 위험 명령 실행 전에 자동으로 검토합니다.
비활성화: /careful --off
```

위험 명령이 감지되면:
```
🔴 위험 명령 감지: [명령]

위험도: [상/중/하]
위험 요소: [구체적인 위험 설명]
영향 범위: [어떤 데이터/시스템에 영향을 주는가]

대안:
  [더 안전한 대안 명령 또는 접근 방식]

계속 실행하시겠습니까? (yes/no)
```

`/careful --off`가 입력되면 세션 모드를 비활성화한다.
```
✅ /careful 모드 비활성화
```

---

## 모드 2: 단일 명령 검토 (`/careful [명령]`)

입력된 명령을 즉시 분석하고 위험도 평가 리포트를 출력한다.
실행은 하지 않는다.

```
🔍 명령 검토: [명령]

위험도: [상/중/하]
위험 요소: [구체적인 위험 설명]
영향 범위: [어떤 데이터/시스템에 영향을 주는가]
되돌리기 가능 여부: [가능/불가능/조건부 가능]

대안:
  [더 안전한 대안 명령 또는 접근 방식]

판정: [실행 권장 / 주의 후 실행 / 실행 비권장]
```

---

## 위험 명령 목록

| 명령 패턴 | 위험도 | 주요 위험 |
|-----------|--------|-----------|
| `rm -rf` | 상 | 복구 불가능한 파일 삭제 |
| `DROP TABLE` | 상 | 테이블 및 데이터 영구 삭제 |
| `DELETE FROM` (WHERE 없음) | 상 | 전체 행 삭제 |
| `git push --force` | 상 | 원격 히스토리 덮어쓰기 |
| `git reset --hard` | 중 | 로컬 변경사항 영구 손실 |
| `git rebase` (공유 브랜치) | 중 | 팀원 히스토리 충돌 유발 |
| secrets/credential 파일 수정 | 중 | 인증 정보 노출/파손 |

## 예시

### /careful
세션 모드 활성화. 이후 대화에서 위험 명령 자동 감지.

### /careful git push --force origin main
```
🔍 명령 검토: git push --force origin main

위험도: 상
위험 요소: main 브랜치의 원격 히스토리를 로컬로 덮어씁니다.
           다른 팀원이 push한 커밋이 사라질 수 있습니다.
영향 범위: 원격 main 브랜치 전체, 팀 전원
되돌리기 가능 여부: 조건부 가능 (reflog로 복구, 단 시간 제한 있음)

대안:
  git push --force-with-lease origin main
  → 다른 사람의 변경사항이 있으면 push를 거부해 안전합니다

판정: 실행 비권장 (--force-with-lease 사용 권장)
```

### /careful rm -rf ./dist
```
🔍 명령 검토: rm -rf ./dist

위험도: 하
위험 요소: dist 디렉토리와 하위 파일을 모두 삭제합니다.
영향 범위: ./dist 디렉토리 (빌드 산출물)
되돌리기 가능 여부: 불가능 (단, 재빌드로 복구 가능)

대안:
  빌드 전 자동 삭제라면 package.json의 prebuild 스크립트에 추가 권장

판정: 주의 후 실행 (빌드 산출물이므로 재생성 가능)
```
